Sistemas de Gerenciamento de Pacotes

Gerenciadores de pacotes instalam e controlam os diferentes pacotes de software (e suas verses) que voc usa em um ambiente. H vrias opes para escolher, por exemplo, Yum, Zypper, dpkg, e Nix (que sero mencionados na seo Vestimenta). Vamos focar no Conda, que tem vrias funcionalidades teis.

O que Conda faz?

Conda permite que os usurios criem qualquer nmero de ambientes inteiramente separados, e rapidamente e alternam entre eles. Por exemplo, digamos que um pesquisador tem um projeto, Projeto Um, que tem seu prprio ambiente, definido pela Conda, composto do seguinte conjunto de pacotes:

Nome do Pacote

Verso

Pacote A

1.5.2

Pacote B

2.1.10

Pacote C

0.7.9

Mais tarde, o pesquisador comea o Projeto Dois em seu prprio ambiente, com os seguintes pacotes:

Nome do Pacote

Verso

Pacote B

2.1.10

Pacote C

1.2.4

Pacote D

1.5.2

Pacote E

3.7.1

Note aqui que a verso do pacote C usada no Projeto Dois foi atualizada a partir da verso utilizada no Projeto Um. Se esses ambientes de projeto no fossem separados, o pesquisador teria a escolha de:

  • A) Usando a verso mais antiga do pacote C para sempre e no se beneficiando com atualizaes e correes de bugs em verses posteriores.

  • B) Instalando a verso atualizada do pacote e esperando que ele no impacte o Projeto Um.

  • C) Instalando a verso atualizada do pacote para uso no Projeto 2, depois desinstal-lo e reinstalar o antigo sempre que eles precisarem fazer trabalho no Projeto Um. Isso seria extremamente aborrecido e arrisca-se a ser esquecido.

Todas estas opes so extremamente fracas, da a utilidade da Conda para criar ambientes distintos que so facilmente intercambiveis.

Conda tambm pode ser usada para capturar e exportar ambientes computacionais facilmente. Ele tambm pode ir na outra direo; pode gerar ambientes computacionais a partir de arquivos de configurao que podem ser usados para recriar o ambiente de outra pessoa.

Outro benefcio da Conda que ela oferece muito maior flexibilidade aos usurios que no tm privilgios de administrao nas mquinas em que esto trabalhando (como muito comum quando trabalham com instalaes de computao de alto desempenho). Sem a Conda, tipicamente desafiador instalar o software exigido em tais mquinas. No entanto, porque Conda cria e altera novos ambientes, ao invs de fazer alteraes no ambiente geral de uma mquina, os privilgios de administrador no so necessrios.

Finalmente, embora o Conda seja, at certo ponto, centrado em Python, tambm est bem integrado para uso com outras lnguas. Por exemplo, a verso base do Conda inclui a biblioteca padro de C++.

Instalando Conda

Note que estas instrues de instalao so direcionadas para sistemas Linux. Instrues para instalar o Conda no Windows ou Mac system podem ser encontradas aqui.

V para https://repo.continuum.io/miniconda/ e baixe o ltimo instalador do Miniconda 3 para o seu sistema (32 bits ou 64 bits). Ser um nome como miniconda_version_number.sh. Execute o instalador usando:

bash Miniconda_verso_nmero.sh

Voc pode verificar se o Conda foi instalado com sucesso digitando:

conda --version

que deve retornar um nmero de verso.

Criao e Uso de Ambientes

Conda instala automaticamente um ambiente de base com alguns pacotes de software comumente usados. possvel trabalhar neste ambiente de base; no entanto, uma boa prtica criar um novo ambiente para todos os projectos que se inicia.

Para criar um ambiente, utilize conda create --name your_project_env_name seguido por uma lista de pacotes a incluir. Para incluir os pacotes scipy e matplotlib , adicione-os ao final do comando:

criar conda --name Projetot_One scipy matplotlib

Pode especificar as verses de certos (ou todos) pacotes usando =package_number aps o nome. Por exemplo, para especificar scipy 1.2.1 no ambiente acima:

criar conda --name Projetot_One scipy=1.2.1 "Ttulo"

Ao criar ambientes, voc tambm pode especificar verses de idiomas para instalar. Por exemplo, para usar o Python 3.7.1 no ambiente Project_One:

conda create --name Project_One python=3.7.1 scipy=1.2.1 matplotlib

Agora que um ambiente foi criado, hora de ativar (comear a usar) atravs de conda activate environment_name. Portanto, neste exemplo:

conda ativar Projeto_One

Note que voc pode precisar usar a fonte `` em vez de conda se voc estiver usando uma verso antiga do Conda.

Uma vez que um ambiente ativado, voc dever ver o nome do ambiente antes de cada prompt no seu terminal:

(Project_One) $ python --version
Python 3.7.1

Desativando e excluindo ambientes

Voc pode desativar (sair da) um ambiente usando:

"conda" desativar

e remover (excluir) um ambiente como mostrado aqui:

remover conda env --name Projeto_One

Para verificar se um ambiente foi removido com sucesso, voc pode ver uma lista de todos os ambientes de Conda no sistema utilizando:

lista env conda

No entanto, a excluso de um ambiente pode no apagar os arquivos do pacote que foram associados a ele. Isto pode levar a que muita memria seja desperdiada em pacotes que j no so necessrios. Pacotes que no so mais referenciados por nenhum ambiente podem ser excludos usando:

"conda limpo" -pts

Como alternativa, voc pode excluir um ambiente (como Project_One), juntamente com seus pacotes associados via:

'conda' remover --name Projeto_One --todos

Instalar e remover pacotes dentro de um ambiente

Dentro de um ambiente, voc pode instalar mais pacotes usando:

instalar o package_name conda

Da mesma forma, voc pode remov-los via:

apagar nome do pacote

Esta a melhor maneira de instalar pacotes de dentro do Conda, pois tambm instalar uma verso adaptada Conda do pacote. No entanto, possvel utilizar outros mtodos se uma verso especfica do pacote no estiver disponvel. Por exemplo, pip comumente usado para instalar pacotes de Python. Ento, um comando como:

pip install scipy

ir instalar o pacote scipy explicitamente - desde que o pip seja instalado dentro do ambiente de Conda atualmente ativo. Infelizmente, quando Conda e pip so usados juntos para criar um ambiente, pode conduzir a um estado que pode ser difcil de reproduzir. Especificamente, executar Conda depois do pip pode potencialmente substituir ou quebrar pacotes instalados via pip. Uma maneira de evitar isso instalando o maior nmero possvel de requisitos no Conda, e ento usando o pip. Informaes detalhadas podem ser lidas no post, Usando o Pip em um ambiente de Conda.

Embora os pacotes do Python tenham sido usados em muitos dos exemplos dados aqui, os pacotes do Conda no precisam ser pacotes do Python. Por exemplo, aqui o idioma base R est instalado juntamente com o pacote R r-yaml:

criar conda --name Projetot_One r-yaml

Para ver todos os pacotes instalados no ambiente atual, use:

lista de conda

Para verificar se um pacote em particular est instalado, por exemplo, a scipy neste caso:

conda list scipy

Um canal de Conda de onde ele baixou um pacote. Canais comuns incluem Anaconda (uma empresa que fornece o canal de pacote de conda padronizado), e conda-forge (uma tentativa de embalagem conduzida pela comunidade). Voc pode instalar explicitamente um pacote a partir de um determinado canal, especificando-o como:

conda install -c channel_name package_name

Exportando e Reproduzindo Ambientes Computacionais

Ambientes de Conda podem ser exportados facilmente para arquivos legveis por humanos no formato YAML. Arquivos YAML so discutidos com mais detalhes mais tarde neste captulo.

Para exportar um ambiente conda para um arquivo chamado environment.yml, ative o ambiente e ento execute:

exportao de env conda > environment.yml

Da mesma forma, ambientes de Conda podem ser criados a partir de arquivos YAML via:

conda env criar -f environment.yml

Isso permite que pesquisadores se reproduzam rapidamente em ambientes computacionais uns dos outros. Note que a lista de pacotes no apenas aqueles explicitamente instalados. Ele pode incluir pacotes de dependncias especficas para Sistema Operacional para que os arquivos de ambiente possam exigir alguma edio para ser porttil para diferentes sistemas operacionais.

Os ambientes tambm podem ser clonados. Isso pode ser desejvel, por exemplo, se um investigador iniciar um novo projecto e quiser criar um novo ambiente para trabalhar nele; o ambiente do novo projeto (pelo menos inicialmente) pode exigir os mesmos pacotes do ambiente de um projeto anterior.

Por exemplo, para clonar o ambiente Project_One e dar a este novo ambiente o nome Project_Two:

criar conda --name Projeto_Dois --clonar Projeto_Um
