Gerenciamento de dados pessoais

Esta seo fornece uma viso geral das coisas a considerar ao trabalhar com dados pessoais. Para uma viso mais prtica sobre ferramentas e prticas que facilitam a reproduo, por favor veja o captulo Projetos de Dados Sensveis.

Dados pessoais

Dados pessoais so informaes sobre pessoas vivas que podem ser identificadas usando os dados que voc est processando, seja direta ou indiretamente (por exemplo, o nome de uma pessoa, endereo ou outro identificador exclusivo, como seu nmero de Segurana Social). Dados relacionados ao falecido no so considerados dados pessoais na maioria dos casos, sob o RGPD. Os identificadores indirectos incluem a sade, as caractersticas econmicas, culturais ou sociais. Especialmente quando uma determinada combinao de identificadores pode ser utilizada para identificar uma pessoa, h que ter o cuidado de gerir devidamente os dados. Dados particularmente sensveis incluem dados relacionados com a pessoa:

  • identidade racial/tnica

  • opinies polticas

  • Crenas religiosas/filosficas

  • Associao sindical

  • Dados genticos e biomtricos

  • sade fsica ou mental

  • orientao sexual

Polticas de dados pessoais

Existem vrias polticas em diferentes pases para proteger os direitos dos indivduos sobre os seus dados pessoais. Por exemplo, os dados pessoais na Austrlia esto regulamentados nos termos da Lei de Privacidade australiana. Na Unio Europeia GDPR (Regulamento Geral de Proteo de Dados) se aplica ao processamento de dados pessoais e pode exigir que voc leve a cabo uma Avaliao de Impacto de Proteo de Dados (DPIA). Processar significa fazer qualquer coisa com as informaes de uma pessoa, incluindo coleta, armazenamento, anlise, compartilhamento, excluso e destruio. Para garantir que voc est atualizado com os requisitos de gerenciamento de dados sensveis, por favor, reveja as polticas nacionais/institucionais que se aplicam sua pesquisa. Veja [] para prticas recomendadas de compartilhamento de dados clnicos de teste.

Consentimento informado

O consentimento informado, voluntrio e justo para participar num estudo muito importante para qualquer projecto de pesquisa que envolva participantes humanos. atravs deste processo de consentimento que os participantes da pesquisa podem entender o que participar num estudo especfico significar para eles. Em seguida, cada pessoa pode escolher se deseja participar usando o formulrio de consentimento. Consulte tambm o Guia para Pesquisa tica.

Observe que o formulrio de consentimento informado considerado dados pessoais e, portanto, deve ser tratado com o mesmo cuidado que outros dados pessoais. No armazene os formulrios de consentimento onde armazena os restantes dados que coleta; use um painel de bloqueio separado ou uma pasta criptografada, por exemplo.

Caso no possa usar um formulrio de consentimento por escrito, tente fazer um registo de consentimento verbal.

A documentao do consentimento deve incluir:

  • folha de informaes de um participante e

  • um formulrio de consentimento assinado pelo participante.

The participant information sheet is used to inform participants about the study. A informao deve ser clara e fcil de compreender e deve abranger o seguinte:

  • Do que se trata o projecto.

  • O que implicar a sua participao.

  • Quaisquer riscos envolvidos para os participantes e salvaguardas para minimizar esses riscos.

  • Garantias sobre segurana de dados e confidencialidade dos usurios.

    • Mencione quem tem acesso aos dados

  • Como os dados sero utilizados no estudo (para artigos, relatrios e apresentaes publicadas).

  • Proposta de planos para arquivar dados no final do estudo e potencial reutilizao secundria de dados.

    • Consentimento determinado pode ser uma soluo aqui, permitindo que os participantes escolham que tipo de informao ser compartilhada publicamente

  • Detalhes da organizao que supervisiona a investigao.

  • Quem entrar em contato para obter mais informaes sobre o estudo.

O formulrio de consentimento usado para verificar se o participante da pesquisa entende e concorda em participar do estudo. A forma de consentimento deve cobrir os seguintes pontos no mnimo:

  • O usurio

    • leu e entendeu a folha de informaes do usurio

    • foi dada a oportunidade de fazer perguntas

    • entende que a participao voluntria

    • entende que eles podem se retirar do estudo a qualquer momento, sem dar razes e sem penalizar

    • entende como os dados sero gerenciados, compartilhados e arquivados (como detalhado na folha de informaes)

      • para aumentar a chance de reutilizao dos seus dados, no promete excluir os dados, mas em vez disso pea o consentimento para manter e compartilhar os dados (veja [])

  • Assinaturas tanto do participante como do pesquisador e da data de assinatura

Pense em frente e planeje como voc vai:

  • colete, armazene e gerencie os dados (veja Armazenamento de dados e organizao)

  • controlar permisses de acesso

  • preparar dados para arquivar/compartilhar no final do projeto, se possvel, (veja Dados de compartilhamento e arquivamento)

Existem vrias estratgias que voc pode adotar para proteger a privacidade dos indivduos de sua pesquisa:

1. Minimizao de dados

  • Se as informaes pessoais no forem necessrias, no colete.

  • Revisar periodicamente se voc mantm informaes desnecessrias de identificao.

  • Ao identificar informaes no mais necessrio, remova-as, exclua-as ou destrua-as com segurana.

2. Limites de reteno de dados

  • Decida quanto tempo voc manter dados identificveis antes de remover identificadores diretos, aplicando tcnicas de anonimato mais complexas, ou excluindo todos os dados.

  • Ao excluir dados confidenciais, voc precisa estar ciente de que mtodos padro para excluir arquivos (por exemplo, mover arquivos para a lixeira e esvazi-la) no so seguros. Estes arquivos excludos podem ser recuperados. Use um software como BleachBit para excluir os dados com segurana.

3. Transferncia de dados segura

  • Antes de decidir transferir dados pessoais, voc deve considerar se a transferncia de dados identificveis necessria. Por exemplo, os dados podem ser desidentificados ou anonimizados?

  • Se os dados no podem ser identificados ento voc deve garantir que tem autoridade para transferir os dados pessoais, e que existem salvaguardas adequadas para proteger os dados antes, durante e aps o trnsito.

  • Muitas vezes, a sua universidade ou instituto proporcionar solues para uma transferncia segura de arquivos. Entre em contato com voc pesquisando dados, privacidade ou equipe de suporte de TI para orientao.

4. Encriptao

  • A criptografia fornece proteo garantindo que apenas uma pessoa com a chave de criptografia relevante (ou senha) possa acessar o contedo.

    • Proteger no nvel de disco: Bitlocker para Windows, FileVault para MacOS

    • Proteger no nvel de container (uma pasta contendo vrios arquivos): Veracrypt (ou Arquivo para MacOS)

    • Armazenamento porttil: Bitlocker

    • Nvel de arquivos / Informaes de troca:

      • Mtodo simples: use 7zip e pacote com uma senha

      • Mais complicado de configurar: use ferramentas PGP (tambm pode ser usado para enviar e-mails com segurana)

    • Veja o Manual de Criptografia da Universidade de Gand para Pesquisadores para mais detalhes e guias passo a passo

5. Permisses de acesso

  • Gerenciamento de permisses de pastas compartilhadas.

  • Proteo com senha.

6. Anonymisation

A anonimizao um processo pelo qual a identificao de informaes em um conjunto de dados removida. utilizada, em primeiro lugar, para permitir que dados sejam partilhados ou publicados sem revelar as informaes confidenciais que contm, limitando ao mesmo tempo a perda de informao.

  • Sempre que possvel, identificadores diretos (como nomes, endereos, Os nmeros de telefone e de conta) devem ser removidos assim que as informaes de identificao deixarem de ser necessrias. Voc pode excluir os dados ou substitu-los por pseudnimos. Para dados qualitativos voc deve substituir ou generalizar caractersticas de identificao ao transcrever entrevistas.

  • Dados identificados de-identificados que podem ser reidentificados usando um arquivo de link (por exemplo, informao associando indivduos de dados a indivduos identificveis) conhecido como dados pseudoerrados. NOTA: Neste caso, o arquivo de ligao deve ser criptografado e armazenado de forma segura e separada dos dados de pesquisa no identificados.

    • A identificao de indivduos em dados pseudnimos ou no identificados pode ainda ser possvel usando combinaes de identificadores indiretos (como a idade, educao, emprego, rea geogrfica e condies mdicas). Alm disso, os dados e sadas contendo pequenas contagens de clulas podem ser potencialmente divulgados, particularmente nos casos em que as amostras so extradas de pequenas populaes ou em que incluem casos com valores extremos ou caractersticas relativamente raras.

    • Como tal, quando pretende compartilhar dados potencialmente identificveis ou as sadas geradas a partir dos dados, talvez voc precise considerar tcnicas de anonimato mais avanadas, como controle de divulgao estatstica (SDC, consulte este manual para obter mais informaes).

  • Para obter mais informaes sobre anonimato voc pode assistir este webinar pela Enrico Glerean.

7. Compartilhando dados confidenciais

Se voc planeja compartilhar ou publicar seus dados, deve se certificar de que seus dados so apropriados e seguros para compartilhar. Por exemplo, voc deve considerar se os dados podem ser adequadamente anonimizados, e se os dados annimos permanecero teis (veja tambm Barridores ao compartilhamento de dadosCaptulo de Pesquisa Aberta). Aps aplicar mtodos para desidentificar e anonimizar dados confidenciais, ainda poder haver um risco de re-identificao (veja []).

Uma medida de segurana adicional, ou alternativa anonimizao, est a aplicar controlos de acesso para garantir que os dados sejam partilhados de forma adequada e segura. Isso pode envolver encontrar um repositrio de dados que possa fornecer controles de acesso adequados (veja aqui para uma lista de repositrios de acesso protegidos). Esses repositrios podem fornecer acesso aos metadados do projeto, o que permite que outras pessoas encontrem e o cite os dados. O acesso restrito/condicional tambm fornece potenciais reutilizadores com as informaes que eles precisam para acessar os dados, fazendo com que os dados FALHA. Por exemplo, condies para acessar os dados poderiam exigir o potencial reutilizao de dados para:

  • Registre-se e/ou fornea detalhes de contato para garantir que os reutilizadores sejam genunos e conscientes das suas responsabilidades

  • Fornea informaes sobre como eles usam os dados

  • Concordar com condies (segurana dos dados, privacidade, restries includas nos formulrios de consentimento)
