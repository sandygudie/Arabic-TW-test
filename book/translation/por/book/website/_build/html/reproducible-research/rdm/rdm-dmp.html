Plano de gerenciamento de dados

Um Plano de Gerenciamento de Dados (DMP) ou Plano de Gesto de Sada, um documento que descreve como suas sadas de pesquisa sero geradas, armazenadas, usadas e compartilhadas em seu projeto. Um DMP um documento vivo, que pode ser atualizado em todo o projeto de pesquisa conforme necessrio.

Um Plano de Gerenciamento de Dados um roteiro para voc gerenciar seus dados de forma eficiente e segura. Isso pode impedir a perda ou a violao de dados. Planejando com antecedncia como gerenciar seus dados de forma consistente pode poupar seu tempo mais tarde! Tambm pode tornar mais fcil para compartilhar os seus dados com outras pessoas e, portanto, tornar os dados mais FALHA

Um Plano de Gesto de Dados dever fornecer informaes sobre cinco tpicos principais:

1. Funes e responsabilidades

  • importante discutir quem responsvel por diferentes tarefas durante o ciclo de vida de um projecto de investigao. Definir quem responsvel pela gesto dos dados e cdigo pode evitar confuso/m comunicao mais tarde no projeto.

  • Voc deve verificar as recomendaes do DMP e os requisitos do seu instituto e financiador. A equipe de suporte de pesquisa de biblioteca do seu instituto e o site do seu financiador geralmente so lugares bons para encontrar informaes e ajudar. Alguns dos financiadores exigem que voc use seu modelo DMP. Voc pode verificar se o seu financiador ou instituto tem um DMP usando DMPonline.

2. Tipo e tamanho dos dados coletados e documentao/metadados gerados

  • Aqui voc pode listar os formatos de arquivo que voc usar para coletar, processar e apresentar os seus dados. Se voc quiser compartilhar suas sadas de pesquisa mais tarde Os formatos de arquivo padro que podem ser abertamente usados sem uma licena especfica para um programa de software so preferidos. Para garantir isso, voc deve adaptar seus arquivos ou comear a trabalhar nesses formatos mais cedo.

  • Uma distino pode ser feita entre diferentes tipos de dados que podem ser descritos separadamente no plano:

    • Dados raros/primrios: dados coletados da fonte (mantenha sempre uma verso somente leitura de dados brutos para que voc possa voltar mais tarde!)

    • Dados processados: uma verso dos dados que foram modificados para anlise ou visualizao

    • Dados finalizados: dados que esto prontos para serem compartilhados em uma publicao ou repositrio de dados (veja Compartilhar e arquivar seo de dados para obter mais informaes). Alguns repositrios de dados, como o Zenodo, permitir versionamento de datasets para que voc possa atualizar seu dataset finalizado se quiser liberar outra verso.

  • Todos esses tipos de dados tero de ser descritos para serem colocados no contexto, usando metadados (veja adocumentao e a seo de metadados da {ref}<rr-rdm-metadata>) e a documentao adequada, que permitir o futuro de voc, e qualquer pessoa da sua equipe, para interpretar os dados.

  • til saber o tamanho aproximado (na faixa de MB, GB, TB ou PB) dos dados nestas vrias etapas porque isso afetar as solues de armazenamento disponveis para voc (discutida no prximo ponto).

3. Tipo de armazenamento de dados usado e de backup de procedimentos que esto em vigor

  • Check the data storage and organisation section for storage and back-up solutions and ways to organise your files

  • Verifique se existem custos associados ao seu projeto

    • Soluo de armazenamento preferida (durante e aps o projeto, veja abaixo)

    • Custos pessoais (se voc precisa de um gerente de dados para gerenciar grandes quantidades de dados)

    • Licenas de software (como Notebooks do Electronic Lab, consulte a Seo de notebooks abertos

    • Voc pode usar esta lista de verificao pelos custos como orientao

  • Manter o controle de quem fez alteraes especficas em seus dados/cdigo ser importante, particularmente para o cdigo. Consulte o captulo de Controle de Verso para obter mais informaes.

  • Determine quem tem acesso aos dados e quem concede acesso. Pelo menos uma outra pessoa deve ter acesso aos seus dados, como seu supervisor/PI/chefe do departamento. Se voc estiver gerenciando dados pessoais/comercialmente sensveis, o acesso s deve ser dado a indivduos que tenham que trabalhar com os dados.

4. Preservao dos resultados de pesquisa aps o projeto

  • Considere se suas sadas de pesquisa podem ser disponibilizadas ao pblico. Dados pessoais ou sadas de pesquisa necessrias para solicitar patentes no podem ser compartilhados publicamente, ver a Seo de dados abertos Se os dados no puderem ser disponibilizados publicamente, ainda ter que preserv-los por vrios anos, dependendo das polticas do seu pas, instituto e financiador.

  • possvel externalizar a preservao de seus dados a longo prazo para um repositrio de dados. Voc pode encontrar mais informaes sobre como selecionar um repositrio apropriado no compartilhamento e arquivamento de dados seo

    • Selecione repositrios usando, por exemplo, FAIRsharing ou Lista recomendada da Natureza, que fornece um identificador persistente, como um DOI para a sua sada de pesquisa. Um repositrio deve ter uma poltica de preservao que especifique quanto tempo as suas sadas sero curadas. Em caso de dvida, entre em contato com sua equipe de suporte de pesquisa de dados para obter mais informaes sobre repositrios de dados.

5. Reutilizao de suas sadas de pesquisa por outros

  • Selecione uma licena quando voc tornar seu resultado disponvel em um repositrio (veja os subcaptulos de licena nos dados e software para mais informaes). Selecionando uma licena, voc informa aos outros como podem reutilizar seus dados. Se voc no selecionar uma licena, outros no sero capazes de reutilizar seus dados sem pedir permisso.

  • Voc pode colocar suas sadas de pesquisa em contexto usando e texto de introduo, como um arquivo README.txt

    • Veja a documentaode documentao e metadados do {ref}<rr-rdm-metadata>

Voc pode usar esta lista de verificao para ver se voc tem tudo coberto pelo seu Plano de Gesto de Dados.
