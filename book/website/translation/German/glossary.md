# Glossary

## A

```{glossary}

Acceptance Testing
 Eine Stufe des Software-Testprozesses, bei der ein System auf Akzeptanz getestet wird. Der Zweck dieses Tests ist es, die Einhaltung der Projektanforderungen durch das System zu bewerten und zu beurteilen, ob er für diesen Zweck annehmbar ist.

Danksagungen
 wo Beiträge zu einem Projekt, das nicht als Autorschaft qualifiziert ist, geschrieben werden. Es wird der Name der Beitragenden aufgezeichnet, und der Beitrag, den sie geleistet haben, wird beschrieben.

Füge
 Befehl hinzu, um Dateien zum Staging-Bereich hinzuzufügen. Ermöglicht dem Benutzer festzulegen, welche Dateien oder Verzeichnisse beim nächsten Commit enthalten sollen.

Autoren
  sind die Autoren in diesem Zusammenhang die Mitwirkenden des _The Turing Way_ Projekts, die einen wesentlichen Beitrag zum Projekt geleistet haben, wie zum Beispiel das Schreiben eines Unterkapitels, Unterstützung von Community-Interaktionen, Pflege der Projekt-Infrastruktur und Unterstützung der Teilnahme anderer durch Mentoring-Beiträge. Alle Autoren sind als Mitautoren im Buch als Ganzes benannt.

```

## B

```{glossary}

Binder
  Ein web-basierter Service, der es Benutzern erlaubt, voll funktionierende Versionen ihrer Projekte in einer von ihnen definierten Umgebung hochzuladen und zu teilen.

Binderhub
  Ein Dienst, der Binder generiert. Die am weitesten verbreitete ist mybinder.org, die vom Binder-Team betreut wird. Es ist möglich andere BinderHubs zu erstellen, die spezialisierte Konfigurationen unterstützen können. Eine solche Konfiguration könnte die Authentifizierung beinhalten, damit private Repositories unter engen Mitarbeitern geteilt werden können.

Verbinde
  , um einen Binder eines Projekts zu erstellen.

Zweig
  Eine parallele Version eines Projektarchivs. Obwohl es im selben Projektarchiv enthalten ist, erlaubt es Ihnen, es separat zu entwickeln und dann die Änderungen wieder in das 'live' Projektarchiv oder mit anderen Zweigen zu verschmelzen.

Fehler
  Dies ist ein Fehler, fehlerhaft oder fehlerhaft in einem Computerprogramm oder System, das es zu einem falschen oder unerwarteten Ergebnis führt oder sich ungewollt zu verhalten.

Baue
  eine Gruppe von Jobs. Zum Beispiel könnte eine Build zwei Jobs haben, von denen jeder ein Projekt mit einer anderen Version einer Programmiersprache testet. Ein Bauwerk beendet, wenn alle seine Jobs beendet sind.

```

## C

```{glossary}

Checkout
  Git Befehl, um zu einer bestimmten Datei, Branch oder Commit zu wechseln. Ermöglicht die Aktivierung älterer Versionen von Dateien oder die Übertragung oder den Wechsel zwischen aktiven Zweigen.

Citizen Science
  Die Einbeziehung von Mitgliedern der Öffentlichkeit in die wissenschaftliche Forschung.


  Kopiere ein bestehendes Git-Repository, normalerweise von einem entfernten Ort in deine lokale Umgebung. Wenn Sie einen Repo klonen, kopieren Sie den gesamten Verlauf sowie alle Zweigen.

Code Coverage
  Eine Maßnahme, die beschreibt, wie viel des Quellcodes von der Testsuite ausgeübt wird.

Code of Conduct
  Richtlinien, die die Art des in der Gemeinschaft geförderten Verhaltens begründen skizzieren Sie den Prozess, durch den Probleme oder Verstöße gegen die Leitlinien angegangen werden und wer für deren Durchsetzung zuständig sein wird.

Code Review
  Eine zusätzliche Möglichkeit um Code Qualität zu testen. Code-Rezension bekommt einen anderen Programmierer, um den neuen Code zu betrachten und ihn zu bewerten. Ziel ist es, Stärken und Verbesserungspotenziale aufzuzeigen.

Erzwungene Autorschaft
  wenn ein leitender Forscher einen Nachwuchsforscher zwingt, ein Geschenk oder einen Gastautor zu enthalten.

Übertragen
  Momentaufnahme des Projektverlaufs. Eine Übertragung kann nach Änderungen an einer einzelnen Datei oder einer Reihe von Dateien und Verzeichnissen erfolgen.

Commit Nachricht
  Eine Nachricht, die der Benutzer einem Commit hinzufügen kann, um zu erklären, was sie enthält.

Kommunikationskanal
  Die Kommunikationsmethode für Projekte, die Mailinglisten, Community-Foren, Chats und/oder soziale Medien beinhalten können.

Community-Mitglied
  Leute, die das Projekt verwenden. Sie können in Gesprächen aktiv sein oder ihre Meinung über die Richtung des Projekts äußern.

Rechenumgebung
  Merkmale eines Computers, die das Verhalten der Arbeit daran beeinflussen können wie z.B. sein Betriebssystem, welche Software es installiert hat und welche Versionen von Softwarepaketen installiert sind.

Conda
  Ein häufig verwendetes Paketverwaltungssystem.

Autoren von Konsortien
 Ein kollektives oder gemeinschaftliches Autorenmodell. Alle Mitglieder des Konsortiums werden als Autoren betrachtet und müssen in der Regel in dem veröffentlichten Artikel aufgeführt werden, obwohl manchmal der Artikel im Gruppennamen veröffentlicht wird. Wenn nicht alle Mitglieder des Konsortiums der Verantwortung der Urheberschaft zustimmen die Mitglieder, die Autoren sind, werden getrennt von denen, die nicht aufgeführt werden.

Container
  Leichte Dateien, die eine komplette Rechenumgebung inklusive Betriebssystem, angepasste Einstellungen, Software und Dateien zusammenfassen können.

Kontinuierliche Zustellung
  automatisiert und führt die Schritte aus, die erforderlich sind, um ein Projekt zu erstellen und zu testen.

Kontinuierliche Verteilung
  Bei jeder Änderung eines Codes wird automatisch bereitgestellt.

Kontinuierliche Integration
  ist die Praxis, Änderungen an einem Projekt von Einzelpersonen in ein Hauptprojekt zu integrieren, häufig geteilte Version (meist mehrmals pro Tag). Auch CI genannt.

Richtlinien mit
  Richtlinien beschreiben, wie eine Person zu einem Open-Source-Projekt beitragen soll.

Mitwirkende
  Jeder, der etwas zum Projekt beigetragen hat. Sie sind Mitglieder eines Forschungsprojekts, das eine Arbeit geleistet hat, die einen Beitrag zur allgemeinen Vollendung der Forschung geleistet hat. Dies könnte ein kleiner Beitrag sein, wie zum Beispiel die Behebung eines Fehlers in Software oder ein wesentlich größerer Beitrag wie das Schreiben eines akademischen Artikels.

Korrespondierender Autor
 Die Person, die einen akademischen Artikel für die Forschungsgruppe verwaltet. Sie sind dafür verantwortlich, die Kommentare der Überprüfer, die Beweise zu erhalten, mit den Redakteuren und deren Angaben werden auf der finalen Fassung des veröffentlichten Artikels gedruckt.

KRediT Taxonomie
 Die KRediT Taxonomie ist eine hochrangige Taxonomie, einschließlich 14 Rollen, , die benutzt werden kann, um die Rollen zu repräsentieren, die normalerweise von Mitwirkenden an wissenschaftlicher wissenschaftlicher Leistung gespielt werden. Die Rollen beschreiben den spezifischen Beitrag jedes Beitragenden zur wissenschaftlichen Leistung. Diese Details werden zunehmend von Zeitschriften und Autoren verlangt, die die Urheberkriterien erfüllen.
```

## D

```{glossary}

Datendepot
  Siehe Repository.

DMP
  Datenverwaltungsplan.

Docker Container
  Eine aktive Rechenumgebung, die von einem Docker Image ausgeführt wird.

Dockerfile
  Eine Datei, die zum Erstellen von Docker-Bildern verwendet wird

Docker Image
  Eine maschinenlesbare Anleitung, um eine bestimmte Rechenumgebung zu erstellen.

Docker Registry
  Ein Speicher- und Distributionssystem für benannte Docker Bilder. Die Registrierungsstelle erlaubt Docker-Benutzern, Bilder lokal zu ziehen, sowie neue Bilder in die Registry zu schieben (sofern zutreffend entsprechende Zugriffsberechtigungen). Solche Systeme werden oft in der Cloud gehostet, um leicht zugänglich zu sein.

Digital Object Identifier
  Ein digitaler Objektbezeichner (DOI) ist ein persistenter Bezeichner oder Handle zur eindeutigen Identifizierung von Objekten standardisiert durch die International Organization for Standardization (ISO). Eine Implementierung des Handle-Systems, DOIs werden hauptsächlich zur Identifizierung akademischer, professioneller DOIs verwendet und Regierungsinformationen, wie Zeitschriftenartikel, Forschungsberichte, Datensätze und offizielle Publikationen. Sie wurden jedoch auch verwendet, um andere Arten von Informationsressourcen zu identifizieren, wie z. B. kommerzielle Videos.
```

## G

```{glossary}
Verallgemeinerbar
  Durch das Kombinieren von replizierbaren und robusten Erkenntnissen können wir generalisierbare Ergebnisse erzielen. Beachten Sie, dass die Durchführung einer Analyse auf einer anderen Software-Implementierung und mit einem anderen Datensatz keine generalisierten Ergebnisse liefert. Es wird noch viele weitere Schritte geben, um zu wissen, wie gut die Arbeit auf alle verschiedenen Aspekte der Forschungsfrage zutrifft. Generalisierung ist ein wichtiger Schritt zum Verständnis, dass das Ergebnis weder von einem bestimmten Datensatz noch von einer bestimmten Version der Analyse-Pipeline abhängig ist.

Git
  Versionsverwaltungssystem, das GitHub umgebaut wurde. Es ist ein weit verbreitetes Open-Source-verteiltes Versionskontrollsystem, das vom Autor von Linux entwickelt wurde.

Github
  Ein Online-Code-Hosting- und Versionskontrolldienst. Es hat eine Vielzahl von Funktionen, um die Zusammenarbeit zwischen Benutzern zu unterstützen und beherbergt eine große Anzahl von Open-Source-Projekten.

GitLab
  GitLab ist ein web-basiertes DevOps Lebenszyklus-Tool, das einen Git-Repository-Manager zur Verfügung stellt, der Wiki bereitstellt Issue-Tracking und kontinuierliche Integration und Deployment-Pipeline Features, unter Verwendung einer Open-Source-Lizenz, entwickelt von GitLab Inc.

Geisterautor
 Es ist eine Person, die einen akademischen Artikel schreibt, ohne die Forschung durchgeführt zu haben. Es könnte ein professioneller Schriftsteller sein. Sie würden oft nicht als Autor nach den ICMJE-Kriterien für die Autorisierung qualifiziert.

Geschenk-Autor
 Leute, die als Autoren gelistet sind, aber keine wesentlichen Beiträge zur Forschung geleistet haben. Dies ist auch als Gastautor bekannt.

Gruppen-Autor
 Einige Zeitschriften erlauben die Verwendung von Gruppennamen, aber viele verlangen, dass Mitwirkende aufgelistet und/oder die Schreibgruppe benannt werden. Das ist das Gleiche wie die gemeinsame Autorschaft.

Garant
 sowie Erfüllung von Kriterien für den benannten Autor, einige Zeitschriften benötigen einen oder mehrere Autoren, die die Verantwortung für die Integrität der Arbeit als Ganzes von Beginn bis zum veröffentlichten Artikel übernehmen.
```

```{glossary}
Open Educational Resources
```

```{glossary}
  Ein Tool zum Installieren, Verwalten und Deinstallieren von Softwarepaketen inklusive bestimmter Versionen.

Persistent Identifier

Persona Leinwand
  Die Persona Leinwand kann verwendet werden, um alle deine Antworten an einem Ort zusammenzufügen teilen Sie diese greifbare Information Ihres geistigen Modells (abstrakte Konzepte aus unseren Gedanken) mit Ihren Kollegen und erstellen Sie eine gemeinsame Sprache, um über Ihre Mitglieder zu kommunizieren Benutzer und Mitwirkende

Positionalität
 Unterschiede in sozialer Position und Macht prägen Identitäten und den Zugang zur Gesellschaft. Indem wir die Positionalität anerkennen, erkennen wir auch die Überschneidung sozialer Standorte und die komplexe Machtdynamik (auch siehe: Intersektionalität).

Vorabdruck

Es wird normalerweise von den Autoren auf einen öffentlichen Server hochgeladen, auf dem es öffentlich verfügbar ist.

Voraussetzung
  Die Voraussetzung(en) einer Regel entsprechen Dateien oder anderen Zielen im Makefile, die aktualisiert sein müssen, bevor die Regel ausgeführt wird.

Projekt Design
  Eine frühe Phase des Projekts, in der die wichtigsten Merkmale, Struktur, Erfolgskriterien und wichtige Ergebnisse geplant sind.

Pull-Request
  schlug Änderungen an einem Remote-Repository vor. Mitarbeiter ohne Schreibzugriff können einen Pull-Request an den Administrator mit den Änderungen schicken, die sie an der Repo vorgenommen haben. Der Administrator kann dann die Änderungen am Hauptarchiv genehmigen und zusammenführen oder ablehnen. Für Open-Source-Projekte können Pull-Requests von jedem gesendet werden, der ein Projekt abgeforkt hat.

Push
  Änderungen an ein Remote-Repo senden. Das entfernte Repository wird mit den Änderungen aktualisiert und spiegeln nun die lokale Repo.
```

## L

```{glossary}

RDM
  Abkürzung für Forschungsdatenmanagement - siehe Forschungsdatenmanagement zur Definition.

README
  Eine Datei, die nützliche Informationen über ein Projekt wie das was es ist enthält wie man es verwendet/installiert, wie man es testet und wie man dazu beiträgt.

Rezept
  Ein oder mehrere Shell-Befehle, die von Make ausgeführt werden. Normalerweise aktualisieren diese Befehle das Ziel der Regel.

Regressionstest
  Vergleicht das Ergebnis eines Tests vor und nach der Änderung des Codes. Wenn sich die Ausgabe geändert hat, wurde irgendwo ein Problem in das Programm eingeführt, und ein Fehler wird geworfen.

Nachweisbar
  Ein Ergebnis ist replizierbar, wenn die gleiche Analyse auf verschiedenen Datensätzen qualitativ ähnliche Antworten erzeugt.

repo2docker
  Ein Werkzeug zum Erstellen von Docker-Bildern aus Code-Repositories.

Repository
  *Wie Daten oder Code Reprository*. Ein langlebiger Ort im Internet, an dem Ressourcen (seien es Daten, Software, Publikationen oder irgendetwas anderes) gespeichert und aufgerufen werden können. Dieses Schlüsselwort wird oft auf „Repo“ gekürzt.

Reproduzierbar
  Ein Ergebnis ist reproduzierbar, wenn dieselben Analyseschritte, die auf demselben Datensatz durchgeführt werden, konsistent dieselbe Antwort erzeugen.

Gerenderte Ausgabe
  So sieht der Text auf einer Online-Seite in Github oder Webseite aus

Research Compendia
  Dies ist eine Sammlung aller digitalen Teile eines Forschungsprojekts, einschließlich Daten, Code, Texte (Protokolle, Berichte, Fragebögen, Metadaten). Die Sammlung wird so entworfen, dass die Wiedergabe aller Ergebnisse direkt nach vorn erfolgt.

Recherchedatenmanagement
  *Acronym: RDM*. Bezieht sich auf die Organisation, Speicherung und Speicherung von Daten, die während eines Forschungsprojekts erstellt wurden. Sie umfasst Erstplanung, tägliche Prozesse sowie langfristige Archivierung und Freigabe. Kürzt auf RDM.

Forschungsethik
  Forschungsethik sind die moralischen Prinzipien, die bestimmen, wie Forscher ihre Arbeit ausführen sollen. Diese Prinzipien werden verwendet, um Forschungsvorschriften auszuarbeiten, die von Gruppen wie Universitätsleitungsgremien, Kommunen oder Regierungen vereinbart wurden. Alle Forscher sollten sich an alle Vorschriften halten, die für ihre Arbeit gelten.

Überprüfen Sie
  Änderungen vorzuschlagen oder um die Übertragung von etwas zu einer bereits erstellten Pull-Anfrage zu bitten.

Risikobewertung
  Dies wird verwendet, um die richtigen nachhaltigen Softwarekonzepte für Ihr Projekt auszuwählen.

Risk Matrix
  Eine Risikomatrix ist eine Möglichkeit, das Geschehen mit dem Interessenten zu quantifizieren. Eine Achse misst die Exposition auf die eine und die andere die Auswirkungen eines Fehlfalls. Je weiter der Ursprung entfernt ist, desto mehr Schutzmaßnahmen sind erforderlich, um das Risiko akzeptabel zu machen.

Roadmapping
  Dies ist die Erstellung einer Roadmap für Ihr Projekt. Es ist ein Leitfaden für die Arbeit, die Sie leisten müssen. Es umfasst Ihre Ziele, Ihre Vision und eine Zeitleiste für Aufgaben.


Robuster
  Ein Ergebnis ist robust, wenn der gleiche Datensatz unterschiedlichen Analyseworkflows unterworfen wird, um die gleiche Forschungsfrage zu beantworten (z.B. eine Pipeline in R und eine in Python geschriebene Pipeline) und eine qualitativ ähnliche oder identische Antwort produziert wird. Robustes Ergebnis zeigt, dass die Arbeit nicht von den Besonderheiten der zur Analyse gewählten Programmiersprache abhängig ist.

Regel
  Ein Element der Makefile, das etwas definiert, das erstellt werden muss, besteht in der Regel aus Zielen, Rezepten und optionalen Voraussetzungen.

Runtime Test
  Tests eingebettet in das Programm, die als Teil des Programms ausgeführt werden.
```

## S

```{glossary}

Selbstarchivierung:
  Eine Publikation oder andere Forschungsergebnisse in einem geeigneten Repository, institutionell oder fachbezogen platzieren nach den möglichen Einschränkungen durch den Herausgeber, zum Beispiel einen Embargo-Zeitraum oder Beschränkungen der zugelassenen Version, die in solchen Archiven hinterlegt werden können.

Selbstreflexion
  Aktivität des Denkens über unsere Gedanken, Gefühle, Gefühle, Verhaltensweisen und die Gründe die ihnen zugrunde liegen könnten. Wenn wir uns die Zeit der Reflexion nehmen, können wir unser Verständnis dafür erweitern, wer wir sind was unsere Werte sind und warum wir denken, fühlen und so handeln. Wenn wir uns selbst reflektieren und uns dessen bewusst werden, was uns treibt, wir können Änderungen leichter vornehmen, die uns dabei helfen, uns selbst zu entwickeln oder unser Leben zu verbessern, einschließlich der Art und Weise, wie wir Forschung betreiben (Quelle: [Berkeley Wellbeing](https://www. erkeleywellbeing.com/what-is-self-reflection.html)).

SHA
  Eindeutige Zeichenkette von Buchstaben, die zur Identifizierung jedes Commit oder jeden Knoten im Projektarchiv verwendet werden.

Gemeinsame Autorschaft
 Einige Zeitschriften erlauben die Verwendung von Gruppennamen, aber viele verlangen, dass Mitwirkende aufgelistet und/oder die Schreibgruppe benannt werden. Dies ist das Gleiche wie die Gruppen-Autorschaft.

Smoke Testing
  Sehr kurze anfängliche Prüfungen, die sicherstellen, dass die grundlegenden Anforderungen für die Ausführung des Projekts eingehalten werden. Wenn diese fehlschlagen, hat es keinen Sinn, weitere Teststufen einzuführen, bis sie behoben sind.

Staged
  Staging der Änderungen, die im nächsten git Commit enthalten sind.

Stochastic Code
  Code, der zwar korrekt ist, aber nicht immer das gleiche Ergebnis ausgibt. Zum Beispiel erzeugt ein Programm, das zehn zufällige Zahlen ausgibt, jedes Mal ein anderes Ergebnis, obwohl es korrekt ist.

Syntax
  Die Struktur der Anweisungen in einer Computersprache.

System Testing
  Eine Stufe des Software-Testprozesses, bei der ein komplettes und integriertes System getestet wird. Der Zweck dieses Tests ist es, zu beurteilen, ob das System als Ganzes die richtigen Ausgänge für die gegebenen Eingaben liefert. Siehe auch Ende bis Ende-Test.
```
